## 1. IP池基础概念

### 1.1 什么是IP池

**IP池（Proxy Pool / IP Pool）** 是一组代理IP的集合，通过统一的调度管理系统，实现IP的获取、分配、轮换和失效处理。在爬虫、数据采集、账号矩阵、广告验证等场景中，IP池是突破单IP请求限制、避免被目标网站封禁的核心基础设施。

**核心价值**：
- 分散请求来源，避免单一IP被封禁
- 突破地域限制，获取特定地区内容
- 提高请求成功率，保障采集稳定性
- 模拟真实用户行为，降低被识别概率

### 1.2 IP池分类

| 类型 | 数据中心IP（DC Proxy） | 住宅IP（Residential Proxy） | 移动IP（Mobile Proxy） |
|------|----------------------|---------------------------|----------------------|
| **来源** | 云服务器/VPS机房 | 真实家庭用户设备 | 移动运营商（4G/5G） |
| **匿名性** | 低（易被识别为机房IP） | 高（伪装成真实用户） | 极高（运营商IP难以封禁） |
| **速度** | 快（服务器带宽） | 中等 | 较慢 |
| **成本** | 低廉 | 中高 | 高 |
| **适用场景** | 大规模数据采集、对匿名要求不高的场景 | 账号注册、价格抓取、AdTech | 高价值账号操作、反-bot严格网站 |
| **寿命** | 几分钟~几天 | 几分钟~几周 | 数小时~数天 |
| **IP库规模** | 数十万~数百万 | 数百万~数千万 | 数十万 |

### 1.3 IP池在爬虫/数据采集中的核心作用

```
请求链路对比：

无IP池（单一IP）：
[爬虫程序] → [目标网站]  → ❌ 快速触发封禁，IP被拉黑

有IP池（代理轮换）：
[爬虫程序] → [代理管理器] → [代理IP A] → [目标网站] → ✅ 分散请求来源
                    ↓
              [代理IP B] → [目标网站] → ✅
                    ↓
              [代理IP C] → [目标网站] → ✅
```

**核心作用详解**：

1. **请求分发**：将大量请求分散到成百上千个IP，降低单个IP的请求频率
2. **地域伪装**：使用特定地区IP获取地域限制内容（如房产、招聘、本地化电商）
3. **失败重试**：某个代理IP失效时自动切换，不中断采集任务
4. **会话保持**：同一业务请求链使用相同IP，维护登录态和Cookies

### 1.4 常见应用场景

| 场景 | IP类型偏好 | 说明 |
|------|-----------|------|
| 电商价格监控 | 住宅IP | 电商平台对数据中心IP检测严格 |
| 社交媒体账号矩阵 | 住宅/移动IP | 需要高匿名性避免账号关联 |
| SEO排名监控 | 数据中心IP | 量大、成本优先 |
| 广告验证（AdTech） | 住宅/移动IP | 需要真实用户视角 |
| 票价采集（机票/演出） | 住宅IP | 高频请求+地域定价 |
| 金融数据采集 | 数据中心/住宅混合 | 高可用+高匿名 |
| 竞品情报监控 | 视目标网站而定 | 综合成本与成功率 |

## 2. IP池构建技术方案

### 2.1 代理协议

| 协议 | 端口 | 特点 | 适用场景 |
|------|------|------|---------|
| **HTTP/HTTPS** | 80/443/8080 | 明文/加密，最通用 | 大多数Web采集场景 |
| **SOCKS5** | 1080/10808 | 支持TCP/UDP，更灵活 | 协议无关的代理场景 |
| **SOCKS4** | 1080 | 仅支持TCP，无认证 | 简单场景（逐步淘汰） |
| **SSH Tunnel** | 22 | 加密隧道，高匿名 | 少量高安全需求场景 |
| **WireGuard/OpenVPN** | 1194/51820 | VPN协议，完整加密 | 企业级方案 |

**协议选择建议**：
- 大多数Python爬虫场景：HTTP/HTTPS代理
- 需要维持TCP长连接或非HTTP协议：SOCKS5
- 对安全性要求极高：SSH Tunnel或WireGuard

#### 2.2.1 免费代理

**来源**：
- 公开代理网站（如 FreeProxyList、GatherProxy、ProxyScrape）
- GitHub开源代理列表项目
- 社区共享（TG群、论坛）

**致命缺陷**：
- 存活时间极短（平均<5分钟）
- 多人共用，IP早已被各大网站封禁
- 存在安全风险（恶意代理可能记录流量）
- 无法满足正经业务需求

**结论**：仅适合学习研究，绝不适用于生产环境。

#### 2.2.2 付费代理

**按量计费 vs 订阅模式**：

| 计费模式 | 特点 | 适用场景 |
|---------|------|---------|
| **按量计费（Pay-as-you-go）** | 按流量收费，如 $3/GB | 请求量波动大、难以预估 |
| **固定套餐** | 月付固定金额，如 $50/月 | 请求量稳定、可以按月规划 |
| **独享IP** | IP专属，不与他人共用 | 高价值账号操作、严格反-bot |
| **动态IP** | 每次请求自动切换IP | 高频短请求场景 |

#### 2.2.3 自建代理

指自行搭建代理服务器获取IP，是成本最高但可控性最强的方案。详见第3章。

### 2.4 代理轮换策略

代理轮换是IP池的核心调度逻辑，决定了如何从池中选取代理IP进行请求。

#### 2.4.1 轮换策略类型

| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **顺序轮换（Sequential）** | 按顺序依次使用每个IP | 简单，实现容易 | 容易被检测模式 | 少量IP，简单场景 |
| **随机轮换（Random）** | 每次随机选取IP | 难以预测 | 可能重复选中低质量IP | 通用场景 |
| **加权轮换（Weighted）** | 根据IP历史成功率分配权重 | 优先使用高质量IP | 需要持续统计 | 生产环境推荐 |
| **智能切换（Smart）** | 根据目标网站响应动态调整 | 自适应，高效 | 实现复杂 | 大型项目 |
| **会话保持（Sticky）** | 同一会话/任务绑定同一IP | 维持登录态 | 长任务会限制IP切换 | 登录态操作 |
| **地理位置匹配** | IP地理位置与目标网站匹配 | 获取地域正确内容 | IP可用性受限 | 本地化数据采集 |
| **冷却期轮换（Cooldown）** | IP使用后进入冷却期才可再用 | 防过度使用 | IP利用率低 | 防止封禁场景 |

#### 2.4.2 加权随机算法示例

```python
import random

def weighted_random_selection(proxies: list[dict]) -> dict:
    """
    根据代理成功率进行加权随机选择
    proxies: [{"ip": "x.x.x.x", "port": 80, "success_rate": 0.95}, ...]
    """
    # 过滤掉正在冷却的代理
    available = [p for p in proxies if not p.get("in_cooldown", False)]
    
    if not available:
        raise Exception("No available proxies")
    
    # 计算权重（success_rate越高，被选中的概率越大）
    weights = [p.get("success_rate", 0.5) for p in available]
    
    # 加权随机选择
    selected = random.choices(available, weights=weights, k=1)[0]
    return selected
```

#### 2.4.3 智能切换逻辑

```python
def smart_proxy_selector(proxies: list[dict], target_domain: str) -> dict:
    """根据目标网站历史数据智能选择代理"""
    
    # 1. 过滤可用代理（非冷却、非封禁）
    available = [p for p in proxies if p["status"] == "active"]
    
    # 2. 如果该域名最近失败率高，优先选择未使用过该域名的IP
    domain_history = get_domain_history(target_domain)
    
    if domain_history["recent_fail_rate"] > 0.3:
        # 切换策略：选该域名从未失败的IP
        candidate = [p for p in available 
                     if domain_history.get(p["id"], {}).get("failures", 0) == 0]
        if candidate:
            return random.choice(candidate)
    
    # 3. 默认加权随机
    return weighted_random_selection(available)
```

### 2.5 IP质量验证方法

IP质量验证是确保代理池可用性的关键环节，需要持续进行。

#### 2.5.1 验证维度

| 维度 | 检测方法 | 阈值建议 |
|------|---------|---------|
| **可用性** | HTTP请求测试，返回码==200 | 失败连续3次标记失效 |
| **响应时间** | 请求到响应耗时 | >5秒降级，>10秒剔除 |
| **匿名性** | 检测HTTP头中的X-Forwarded-For、Via等 | 存在则标记为透明代理 |
| **被封禁检测** | 访问已知反爬网站测试 | 触发反爬则标记 |
| **地理位置** | 调用IP地理位置API | 与声称不符则标记 |
| **协议支持** | 测试HTTP/HTTPS/SOCKS5支持性 | 不支持则降级 |

#### 2.5.2 异步验证架构

```python
import asyncio
import aiohttp

async def validate_proxy(session: aiohttp.ClientSession, proxy: dict) -> dict:
    """异步验证单个代理"""
    test_url = "http://httpbin.org/ip"  # 可替换为自建验证服务
    
    try:
        start = asyncio.get_event_loop().time()
        async with session.get(test_url, 
                               proxy=f"http://{proxy['ip']}:{proxy['port']}",
                               timeout=aiohttp.ClientTimeout(total=10)) as resp:
            elapsed = asyncio.get_event_loop().time() - start
            if resp.status == 200:
                data = await resp.json()
                return {
                    "proxy": proxy,
                    "status": "valid",
                    "response_time": elapsed,
                    "real_ip": data.get("origin", "")
                }
            else:
                return {"proxy": proxy, "status": "failed", "reason": f"HTTP {resp.status}"}
    except Exception as e:
        return {"proxy": proxy, "status": "error", "reason": str(e)}

async def batch_validate(proxies: list[dict], batch_size: int = 50) -> list[dict]:
    """批量异步验证代理"""
    results = []
    connector = aiohttp.TCPConnector(limit=batch_size)
    
    async with aiohttp.ClientSession(connector=connector) as session:
        tasks = [validate_proxy(session, p) for p in proxies]
        results = await asyncio.gather(*tasks)
    
    return [r for r in results if r["status"] == "valid"]
```

## 3. 自建IP池方案

自建IP池适合有技术能力、长期稳定需求、对成本控制要求高的团队。

### 3.1 方案一：VPS服务器群

**原理**：购买多台VPS服务器，在每台服务器上部署代理服务软件，将所有代理汇总到IP池。

**推荐服务商**：

| 服务商 | 特点 | 单机配置参考 | 成本参考 |
|--------|------|------------|---------|
| **AWS EC2** | 全球覆盖，稳定 | t3.medium $30/月 | 10台 = $300/月 |
| **Vultr** | 按小时计费，部署快 | 2CPU/4GB $20/月 | 灵活 |
| **DigitalOcean** | 简单易用 | 2CPU/4GB $20/月 | 简单 |
| **阿里云ECS** | 国内线路，适合国内采集 | 2CPU/4GB ¥80/月 | ¥800+/月 |
| **腾讯云CVM** | 同上 | 2CPU/4GB ¥70/月 | ¥700+/月 |
| **Hetzner** | 性价比极高（欧洲） | 4CPU/8GB €20/月 | 欧洲业务首选 |

```
              │             │                │             │
```

**优点**：完全可控，成本可控（适合中等规模）
**缺点**：IP数量受限于服务器数量，IP地理分布受服务商限制

### 3.2 方案二：拨号VPS（ADSL/PPPoE）

**原理**：利用家庭宽带或企业专线的动态IP特性，通过不断重新拨号获取新的公网IP。

**适用场景**：需要大量不同IP但IP质量要求不高的场景（如某些国内网站）。

**拨号VPS服务商**：
- **秒切拨号VPS**：提供自动拨号脚本，拨一次换一个IP
- **阿里云ADSL**：历史方案，现已较少
- **传统idc服务商**：提供拨号VPS租赁

**工作流程**：
```
1. 爬虫请求 → Proxy Manager
2. Manager → 向拨号VPS发送指令
3. 拨号VPS执行"断开宽带+重新拨号" → 获得新IP
4. 新IP注册到代理池
5. 爬虫使用新IP访问目标
```

**IP更替频率**：
- 普通ADSL：每次拨号换IP（几十秒~几分钟）
- 优质拨号VPS：可实现秒级IP切换

**成本参考**：¥200-500/月/台，可获得数千~数万IP/天

### 3.3 方案三：云服务商的IP资源

#### 3.3.1 AWS EC2 + Elastic IP

- AWS EC2可以申请多个弹性IP（Elastic IP）
- 按量付费模式下可申请多个EIP
- 配合Auto Scaling可动态创建/销毁实例
- **限制**：AWS对大量EIP使用有风控，需要提前报备

#### 3.3.2 阿里云ECS

- 同一账号可申请多个EIP（按量付费）
- 配合弹性网卡（ENI）可实现多IP
- 国内IP质量高，适合国内采集
- **成本**：EIP ¥0.8/小时/IP（按量）

#### 3.3.3 容器化+IP分配

### 3.4 方案四：树莓派/边缘设备集群

适合有物理设备条件、对IP多样性要求高的团队。

**架构**：
- 在不同地区、不同网络环境下部署树莓派设备
- 每个树莓派运行代理服务
- 通过中心节点统一管理

### 4.4 可用性监控与自动剔除

#### 4.4.1 健康检查机制

```python
import time
from collections import defaultdict

class HealthChecker:
    """代理健康检查器"""
    
    def __init__(self, proxy_manager, config):
        self.pm = proxy_manager
        self.check_interval = config.get("check_interval", 60)  # 秒
        self.max_failures = config.get("max_failures", 3)
        self.max_response_time = config.get("max_response_time_ms", 5000)
        self.failure_counts = defaultdict(int)
        self.last_check = {}
    
    def should_remove(self, proxy: dict) -> tuple[bool, str]:
        """判断代理是否应该被移除"""
        proxy_id = proxy["id"]
        failures = self.failure_counts.get(proxy_id, 0)
        
        # 失败次数超限
        if failures >= self.max_failures:
            return True, f"Excessive failures: {failures}"
        
        # 响应时间超限
        avg_time = proxy.get("avg_response_time_ms", 0)
        if avg_time > self.max_response_time:
            return True, f"Slow response: {avg_time}ms"
        
        return False, ""
    
    def record_result(self, proxy_id: str, success: bool, response_time_ms: int):
        """记录代理请求结果"""
        if success:
            # 成功后重置失败计数
            self.failure_counts[proxy_id] = 0
            self.pm.update_proxy_stats(proxy_id, success=True, 
                                       response_time_ms=response_time_ms)
        else:
            self.failure_counts[proxy_id] += 1
            self.pm.update_proxy_stats(proxy_id, success=False)
    
    def run_continuous(self):
        """持续健康检查循环"""
        while True:
            active_proxies = self.pm.get_active_proxies()
            for proxy in active_proxies:
                should_remove, reason = self.should_remove(proxy)
                if should_remove:
                    self.pm.deactivate_proxy(proxy["id"], reason)
            
            time.sleep(self.check_interval)
```

## 5. 反封禁策略

### 5.1 请求频率控制

**核心原则**：模拟真实用户行为，不超过人类正常操作频率。

```python
import time
import random
from collections import defaultdict

class RateLimiter:
    """多维度频率控制器"""
    
    def __init__(self):
        # 每个IP的请求时间戳记录
        self.ip_request_times = defaultdict(list)
        # 全局限流
        self.global_request_times = []
        
        # 限制配置（建议值，需根据目标网站调整）
        self.limits = {
            "per_ip": {
                "requests_per_minute": 20,      # 单IP每分钟请求数
                "requests_per_hour": 300,        # 单IP每小时请求数
                "cooldown_seconds": 3,            # 请求间隔（秒）
            },
            "global": {
                "requests_per_second": 100,      # 全局每秒请求数
                "requests_per_minute": 3000,     # 全局每分钟请求数
            }
        }
    
    def can_request(self, ip: str) -> tuple[bool, float]:
        """检查是否可以发起请求，返回(允许, 需等待秒数)"""
        now = time.time()
        
        # 1. 检查单IP每分钟限制
        ip_minute_key = int(now / 60)
        recent_1min = [t for t in self.ip_request_times[ip] 
                       if int(t / 60) == ip_minute_key]
        if len(recent_1min) >= self.limits["per_ip"]["requests_per_minute"]:
            wait = 60 - (now % 60) + 1
            return False, wait
        
        # 2. 检查单IP请求间隔（防机器人的核心）
        if recent_1min:
            last_request = max(recent_1min)
            min_interval = self.limits["per_ip"]["cooldown_seconds"]
            elapsed = now - last_request
            if elapsed < min_interval:
                # 加入随机抖动，更像人类
                jitter = random.uniform(0.5, 2.0)
                return False, (min_interval - elapsed) * jitter
        
        # 3. 检查全局每秒限制
        recent_1sec = [t for t in self.global_request_times if now - t < 1]
        if len(recent_1sec) >= self.limits["global"]["requests_per_second"]:
            return False, 1.0
        
        return True, 0
    
    def record_request(self, ip: str):
        """记录一次请求"""
        now = time.time()
        self.ip_request_times[ip].append(now)
        self.global_request_times.append(now)
        
        # 清理过期记录（保留最近10分钟）
        self.ip_request_times[ip] = [t for t in self.ip_request_times[ip] 
                                       if now - t < 600]
        self.global_request_times = [t for t in self.global_request_times 
                                      if now - t < 60]
```

### 5.3 IP冷却策略

当某个IP在目标网站触发封禁后，需要对其进行冷却处理：

```python
from dataclasses import dataclass
from datetime import datetime, timedelta
from typing import Optional

@dataclass
class ProxyCooldown:
    proxy_id: str
    target_domain: str
    blocked_at: datetime
    cooldown_until: datetime
    reason: str

class CooldownManager:
    """IP冷却管理器"""
    
    def __init__(self):
        self.cooldowns: dict[str, ProxyCooldown] = {}
        self.default_cooldown = {
            # 封禁类型 -> 冷却时间（秒）
            "403_forbidden": 300,       # 5分钟
            "429_rate_limit": 600,      # 10分钟
            "captcha": 1800,            # 30分钟
            "ip_ban": 3600,             # 1小时
            "temp_block": 180,          # 3分钟（短时封禁）
        }
    
    def record_block(self, proxy_id: str, target_domain: str, 
                     block_type: str, custom_cooldown: Optional[int] = None):
        """记录一次封禁事件"""
        cooldown_seconds = custom_cooldown or self.default_cooldown.get(
            block_type, 300
        )
        
        # 封禁越严重，冷却时间越长
        if "permanent" in block_type or "ban" in block_type:
            cooldown_seconds = max(cooldown_seconds, 3600)
        
        now = datetime.now()
        self.cooldowns[f"{proxy_id}:{target_domain}"] = ProxyCooldown(
            proxy_id=proxy_id,
            target_domain=target_domain,
            blocked_at=now,
            cooldown_until=now + timedelta(seconds=cooldown_seconds),
            reason=block_type
        )
    
    def is_in_cooldown(self, proxy_id: str, target_domain: str) -> tuple[bool, int]:
        """检查代理是否在冷却中，返回(是否冷却, 剩余秒数)"""
        key = f"{proxy_id}:{target_domain}"
        if key not in self.cooldowns:
            return False, 0
        
        cooldown = self.cooldowns[key]
        now = datetime.now()
        
        if now < cooldown.cooldown_until:
            remaining = (cooldown.cooldown_until - now).seconds
            return True, remaining
        else:
            # 冷却结束，移除记录
            del self.cooldowns[key]
            return False, 0
    
    def get_cooldown_stats(self) -> dict:
        """获取冷却统计"""
        now = datetime.now()
        active = [c for c in self.cooldowns.values() if now < c.cooldown_until]
        return {
            "total_cooldowns": len(self.cooldowns),
            "active_cooldowns": len(active),
            "by_reason": self._count_by_reason(active),
        }
```

### 5.4 行为模式分散

**核心思路**：让爬虫行为尽可能接近真实用户，避免机械化的固定模式。

```python
import random
import time

class BehaviorDisperser:
    """行为模式分散器"""
    
    def __init__(self):
        # 模拟真实用户阅读内容的时间（秒）
        self.reading_times = {
            "short": (5, 15),     # 快速浏览
            "medium": (15, 60),   # 正常阅读
            "long": (60, 180),    # 深度阅读
        }
        
        # 模拟页面滚动行为
        self.scroll_patterns = [
            [0.2, 0.5, 0.8, 1.0],           # 标准深度滚动
            [0.3, 0.6, 0.9, 0.7, 1.0],       # 波浪式滚动
            [0.1, 0.4, 1.0],                 # 浅度滚动
            [0.25, 0.5, 0.75, 0.85, 0.95, 1.0],  # 细致滚动
        ]
    
    def simulate_reading(self, content_length: int) -> float:
        """模拟阅读行为，返回应该停留的秒数"""
        if content_length < 1000:
            category = "short"
        elif content_length < 5000:
            category = "medium"
        else:
            category = "long"
        
        min_time, max_time = self.reading_times[category]
        base_time = random.uniform(min_time, max_time)
        
        # 加入随机抖动（±30%）
        jitter = base_time * random.uniform(-0.3, 0.3)
        return base_time + jitter
    
    def simulate_human_delay(self, action: str) -> float:
        """模拟人类操作之间的随机延迟"""
        delays = {
            "click": (0.5, 2.0),
            "scroll": (0.3, 1.5),
            "search": (1.0, 3.0),
            "page_load": (1.0, 4.0),
            "submit": (0.8, 2.5),
        }
        
        min_d, max_d = delays.get(action, (0.5, 2.0))
        return random.uniform(min_d, max_d)
```

- 设置合理的Crawl-delay（爬虫延迟）

### 7.1 Python代理池核心架构

```python
# proxy_pool.py - 简化的代理池实现

import asyncio
import aiohttp
import random
import time
from dataclasses import dataclass, field
from typing import Optional
from enum import Enum
import redis

class ProxyStatus(Enum):
    ACTIVE = "active"
    COOLDOWN = "cooldown"
    DEAD = "dead"
    CHECKING = "checking"

@dataclass
class Proxy:
    id: str
    ip: str
    port: int
    protocol: str = "http"
    status: ProxyStatus = ProxyStatus.ACTIVE
    success_count: int = 0
    fail_count: int = 0
    total_response_time: float = 0.0
    last_used: Optional[float] = None
    last_checked: Optional[float] = None
    cooldown_until: Optional[float] = None
    tags: dict = field(default_factory=dict)
    
    @property
    def url(self) -> str:
        return f"{self.protocol}://{self.ip}:{self.port}"
    
    @property
    def success_rate(self) -> float:
        total = self.success_count + self.fail_count
        if total == 0:
            return 0.0
        return self.success_count / total
    
    @property
    def avg_response_time(self) -> float:
        if self.success_count == 0:
            return float('inf')
        return self.total_response_time / self.success_count

class ProxyPool:
    """代理池管理器"""
    
    def __init__(self, redis_client: redis.Redis, config: dict):
        self.redis = redis_client
        self.config = config
        self.proxies: dict[str, Proxy] = {}
    
    def add_proxy(self, proxy: Proxy):
        """添加代理到池"""
        self.proxies[proxy.id] = proxy
        self._save_to_redis(proxy)
    
    def get_proxy(self, 
                  protocol: Optional[str] = None,
                  country: Optional[str] = None,
                  require_success_rate: float = 0.0) -> Optional[Proxy]:
        """获取一个可用代理（加权随机选择）"""
        candidates = self._filter_proxies(protocol, country, require_success_rate)
        if not candidates:
            return None
        
        # 加权随机选择（成功率越高，权重越大）
        weights = [p.success_rate for p in candidates]
        selected = random.choices(candidates, weights=weights, k=1)[0]
        
        selected.last_used = time.time()
        self._save_to_redis(selected)
        
        return selected
    
    def report_result(self, proxy_id: str, success: bool, 
                      response_time: float = 0, error: str = ""):
        """上报代理使用结果"""
        proxy = self.proxies.get(proxy_id)
        if not proxy:
            return
        
        if success:
            proxy.success_count += 1
            proxy.total_response_time += response_time
            proxy.status = ProxyStatus.ACTIVE
        else:
            proxy.fail_count += 1
            if proxy.fail_count >= self.config.get("max_failures", 3):
                proxy.status = ProxyStatus.COOLDOWN
                proxy.cooldown_until = time.time() + self.config.get("cooldown_seconds", 300)
        
        self._save_to_redis(proxy)
    
    def _filter_proxies(self, protocol: Optional[str], 
                        country: Optional[str],
                        require_success_rate: float) -> list[Proxy]:
        """过滤可用代理"""
        now = time.time()
        result = []
        
        for proxy in self.proxies.values():
            # 检查状态
            if proxy.status == ProxyStatus.DEAD:
                continue
            if proxy.status == ProxyStatus.COOLDOWN:
                if proxy.cooldown_until and now < proxy.cooldown_until:
                    continue
                else:
                    proxy.status = ProxyStatus.ACTIVE  # 冷却结束，恢复
            
            # 检查协议
            if protocol and proxy.protocol != protocol:
                continue
            
            # 检查国家
            if country and proxy.tags.get("country") != country:
                continue
            
            # 检查成功率
            if proxy.success_rate < require_success_rate:
                continue
            
            result.append(proxy)
        
        return result
    
    def _save_to_redis(self, proxy: Proxy):
        """持久化到Redis"""
        key = f"proxy:{proxy.id}"
        self.redis.hset(key, mapping={
            "ip": proxy.ip,
            "port": str(proxy.port),
            "protocol": proxy.protocol,
            "status": proxy.status.value,
            "success_count": str(proxy.success_count),
            "fail_count": str(proxy.fail_count),
            "last_used": str(proxy.last_used or ""),
        })

# 使用示例
async def main():
    import redis
    r = redis.Redis(host='localhost', port=6379)
    
    pool = ProxyPool(r, {
        "max_failures": 3,
        "cooldown_seconds": 300,
    })
    
    # 添加代理
    pool.add_proxy(Proxy(
        id="proxy_001",
        ip="203.0.113.10",
        port=8080,
        protocol="http",
        tags={"country": "CN"}
    ))
    
    # 获取代理
    proxy = pool.get_proxy(country="CN")
    if proxy:
        print(f"Using proxy: {proxy.url}")
        
        # 使用代理发起请求
        async with aiohttp.ClientSession() as session:
            try:
                async with session.get(
                    "https://httpbin.org/ip",
                    proxy=proxy.url,
                    timeout=aiohttp.ClientTimeout(total=10)
                ) as resp:
                    result = await resp.json()
                    pool.report_result(proxy.id, success=True, 
                                     response_time=resp.total_time)
                    print(f"Success! IP: {result}")
            except Exception as e:
                pool.report_result(proxy.id, success=False, error=str(e))

if __name__ == "__main__":
    asyncio.run(main())
```

### 7.2 异步代理获取与使用

```python
import asyncio
import aiohttp
from typing import Optional
from proxy_pool import ProxyPool

class AsyncProxyClient:
    """异步代理客户端"""
    
    def __init__(self, proxy_pool: ProxyPool, config: dict):
        self.pool = proxy_pool
        self.config = config
        self.session: Optional[aiohttp.ClientSession] = None
    
    async def __aenter__(self):
        connector = aiohttp.TCPConnector(
            limit=self.config.get("max_connections", 100),
            limit_per_host=self.config.get("max_per_host", 10),
        )
        self.session = aiohttp.ClientSession(connector=connector)
        return self
    
    async def __aexit__(self, *args):
        if self.session:
            await self.session.close()
    
    async def fetch(self, url: str, 
                    retries: int = 3,
                    **kwargs) -> Optional[dict]:
        """带重试的异步请求"""
        for attempt in range(retries):
            proxy = self.pool.get_proxy()
            if not proxy:
                await asyncio.sleep(1)
                continue
            
            try:
                async with self.session.get(
                    url,
                    proxy=proxy.url,
                    timeout=aiohttp.ClientTimeout(
                        total=self.config.get("timeout", 30)
                    ),
                    **kwargs
                ) as resp:
                    if resp.status == 200:
                        data = await resp.json()
                        self.pool.report_result(proxy.id, success=True,
                                               response_time=resp.total_time)
                        return data
                    elif resp.status in [403, 429]:
                        # 被封禁，进入冷却
                        self.pool.report_result(proxy.id, success=False,
                                               error=f"HTTP {resp.status}")
                        await asyncio.sleep(self.config.get("backoff", 5))
                    else:
                        self.pool.report_result(proxy.id, success=False,
                                               error=f"HTTP {resp.status}")
            except asyncio.TimeoutError:
                self.pool.report_result(proxy.id, success=False, error="timeout")
            except Exception as e:
                self.pool.report_result(proxy.id, success=False, error=str(e))
        
        return None

# 使用
async def main():
    async with AsyncProxyClient(proxy_pool, config) as client:
        result = await client.fetch("https://httpbin.org/ip")
        print(result)

asyncio.run(main())
```

## 10. 总结与建议

### 10.1 核心结论

1. **IP池是爬虫/数据采集的基础设施**，规模从几台VPS到数百万IP不等
2. **代理类型选择**：大多数场景用住宅IP即可，成本优先用数据中心IP
3. **自建 vs 购买**：中小规模建议购买代理服务，大规模/高定制需求建议自建
4. **管理是关键**：代理池的价值50%在于获取，50%在于持续的管理与优化
5. **合规先行**：技术能力强的同时，法律合规意识必须同步提升
