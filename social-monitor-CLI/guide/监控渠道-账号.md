# 监控渠道-账号清单

> 结构化账号 registry，供 `monitor.yaml` 配置与 ID 解析使用。  
> 最后更新：2026-06-23

## 字段说明

| 字段 | 说明 |
|------|------|
| `canonical_name` | 跨平台标准名，用于去重与 monitor 配置注释 |
| `platform` | 账号平台 |
| `display_name` | 平台侧显示名（可能与 canonical 不同） |
| `account_id` | 采集用 ID（uid / sec_uid / wxid / user_id）；`-` 表示待补 |
| `entity_type` | `official` 官方机构 / `executive` 高管个人 / `sub_brand` 子品牌或职能号 |
| `priority` | P0 每日必跑 / P1 每日 / P2 隔日或周更 / P3 视频号待建能力 |
| `status` | `pending_id` 待补 ID / `unsupported` 当前系统不支持 / `ready` 可入库 |

### 优先级定义

| 级别 | 范围 | 调度建议 |
|------|------|----------|
| **P0** | 跨 3+ 可采平台的官方技术号 | 每日，`monitor-daily.sh` |
| **P1** | 高管个人号、主流子品牌号 | 每日，与 P0 错峰 |
| **P2** | 单平台或职能细分号 | 隔日 / 周更，控制日请求 ≤80 |
| **P3** | 视频号（系统未支持） | 人工跟踪或待 Octopus/新模块 |

## monitor.yaml 映射示例（P0）

补全 `account_id` 后，按 canonical_name 注释写入：

```yaml
bilibili:
  accounts:
    - uid: 123456789          # 长城技术中心
    - uid: 987654321          # TeslaAI
    - uid: 111222333          # 比亚迪DiLink
    # ... P0 其余 UID

douyin:
  accounts:
    - sec_uid: MS4wLjABAAAA...  # 长城技术中心
    - sec_uid: MS4wLjABAAAA...  # TeslaAI
    # ... P0 其余 sec_uid

wechat:
  accounts:
    - wxid: gh_xxxxxxxxxxxx       # 长城技术中心（RSSHub biz）
    # ... P0 其余 wxid

weibo:
  accounts: []                  # 待补 P1 高管 UID

xiaohongshu:
  users:
    - user_id: 5f8a...          # 长城技术中心
    # ... P0 其余 user_id
  topics: []                    # 或按品牌关键词补充
```

验证：

```bash
social-monitor monitor run --task daily --dry-run
```
