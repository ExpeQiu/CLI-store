# 监控范围定义

> 五平台日常监控范围、反爬策略与数据归档约定。

## 一、监控范围

| 平台 | 监控项 | 频率 | 推荐路径 | CLI / 编排 |
|------|--------|------|----------|------------|
| 微博 | 指定账号动态 | 每日 1 次 | **sm** | `fetch weibo --pages 1 --save --diff` |
| 微博 | 日热搜快照 | 每日 1 次 | **sm** | `fetch weibo-trending --date --save` |
| 公众号 | 指定号发文 | 每日 1 次 | **sm**（Octopus 兜底） | `fetch wechat --save --diff` |
| B站 | UP 视频 + 互动 | 每日 1 次 | **sm** | `fetch bilibili --uid --save --diff` |
| B站 | 新视频弹幕词云 | 增量 | **sm** | `fetch bilibili-danmaku --words` |
| B站 | 新视频评论 | 增量 | **sm** | `fetch bilibili-comments` |
| B站 | 直播弹幕 | 开播事件 | **sm** | `fetch live-danmaku` |
| 抖音 | 账号视频 + 互动 | 每日 1 次 | **sm** | `fetch douyin --save --diff` |
| 抖音 | 视频评论 | 增量（可选） | **sm**（可切 Octopus） | `fetch douyin-comments` |
| 抖音 | 直播弹幕 | 开播事件 | **Octopus** | `import octopus` |
| 小红书 | 主题帖 | 每日 1 次 | **sm**（可切 Octopus） | `fetch xiaohongshu-search` |
| 小红书 | 帖评论 | 增量 | **sm**（可切 Octopus） | `fetch xiaohongshu-comments` |

**图例**：`sm` = social-monitor 直采；`Octopus` = 八爪鱼 CLI 模板采集后 `import octopus` 入库。

查看完整路径表：`social-monitor monitor routes`

**不在日常调度内**：知乎热榜、抖音热榜、B站排行榜等广域热榜。

## 二、反爬策略（零成本）

- 零代理：家用/办公单 IP
- 单账号：每平台 1 个小号 Cookie
- 日请求上限：80（`rate_limit.daily_max_requests`）
- 安全模式：`rate_limit.mode: safe` 或 CLI `--safe`（请求间隔 10~20s）
- 遇 403/429：立即停止，不 retry
- 增量：`--save --diff` 减少重复请求

## 三、account_id 约定

| platform | account_id | 含义 |
|----------|------------|------|
| weibo | `{uid}` | 用户动态 |
| weibo | `trending_{YYYY-MM-DD}` | 日热搜快照 |
| wechat | `{wxid}` | 文章 |
| bilibili | `{uid}` | UP 视频 |
| bilibili | `words_{bvid}` | 弹幕词云 |
| bilibili | `comments_{bvid}` | 视频评论 |
| bilibili | `live_{room_id}` | 直播弹幕 |
| douyin | `{sec_uid}` | 用户视频 |
| douyin | `comments_{aweme_id}` | 视频评论 |
| douyin | `live_{room_id}` | Octopus 直播弹幕 |
| xiaohongshu | `topic:{keyword}` | 主题帖 |
| xiaohongshu | `comments_{note_id}` | 评论 |

## 四、配置与调度

- 主配置：`~/.social-monitor/config.yaml`（Cookie、存储、反爬）
- 监控清单：`~/.social-monitor/monitor.yaml`（见项目根 `monitor.yaml.example`）
- 每日任务：`scripts/monitor-daily.sh` 或 `social-monitor monitor run --task daily`
- 直播任务：`scripts/monitor-live.sh` 或 `social-monitor monitor run --task live`

### cron 示例

```cron
# 每日监控（08:00 起错峰，由 monitor run 内部控制）
0 8 * * * /path/to/social-monitor-CLI/scripts/monitor-daily.sh

# 直播开播检测（每 30 分钟）
*/30 * * * * /path/to/social-monitor-CLI/scripts/monitor-live.sh
```

## 五、采集路径与任务切换

### 5.1 推荐路径原则

| 场景 | 推荐 | 回退 |
|------|------|------|
| API/WebSocket 可用 | social-monitor | Octopus（若配置 fallback） |
| 抖音直播弹幕 | Octopus | 无（不降级直采） |
| 小红书/抖音评论反爬重 | 可切 Octopus | social-monitor |

### 5.2 配置切换（monitor.yaml）

```yaml
routes:
  douyin:live_danmaku: octopus          # 固定
  xiaohongshu:topic_search: octopus     # 按需开启
  douyin:comments: octopus

octopus:
  cli: octopus
  templates:
    douyin:live_danmaku: douyin-live-danmaku
    xiaohongshu:topic_search: xhs-keyword-search
```

平台级覆盖写法：`douyin.routes.comments: octopus`

### 5.3 查看与验证

```bash
social-monitor monitor routes          # 当前生效路径
social-monitor monitor run --task daily --dry-run   # 含 route= 标注
```

编排器在 Octopus 失败且路由允许时自动回退 social-monitor，并记录 `fallback` 状态。

### 5.4 Octopus 分工（抖音直播）

1. Octopus 模板 `douyin-live-danmaku` 采集直播弹幕，输出 JSON 数组
2. `social-monitor import octopus --file result.json --platform douyin --type live_danmaku --room-id {id}`
3. social-monitor 不直采抖音直播 WebSocket

### Octopus JSON 格式

```json
[
  {"content": "弹幕文本", "user": "昵称", "timestamp": 1710000000}
]
```

## 六、验收（DoD）

1. `config check all` 已配置渠道通过
2. `monitor run --task daily --dry-run` 输出完整任务列表
3. `monitor run --task daily` 写入 PostgreSQL，可按 platform/account_id 查询
4. 微博热搜可按 `trending_YYYY-MM-DD` 查历史快照
5. 日 HTTP 请求日志 ≤ 80
