## 一、执行摘要

### 1.1 六大平台反爬强度对比

| 平台 | 反爬强度 | 技术投入 | 维护成本 | 推荐方案 |
|------|---------|---------|---------|---------|
| 微博 | 🔴 极高 | 高 | 高 | Cookie池 + 住宅代理 |
| 微信公众号 | 🔴 高 | 中高 | 中 | 搜狗搜索 + Cookie池 |
| B站 | 🟡 中 | 中 | 低 | API签名 + 代理池 |
| 小红书 | 🔴 极高 | 极高 | 高 | 浏览器自动化 |
| 抖音/快手 | 🔴 极高 | 极高 | 高 | SDK模拟 + 分布式 |
| 知乎 | 🟠 中高 | 中 | 中 | API申请 + Token轮换 |

### 1.2 关键建议

1. **优先搞定微博/微信 Cookie** — 这是 EagleEye 目前最大的卡点
2. **小红书需要单独方案** — JS混淆复杂度最高，建议 Phase 3 再做
3. **代理IP池必须上** — 住宅代理 > 数据中心代理，B站已能识别数据中心IP
4. **合规红线** — 不碰 Twitter/X（美国法律风险）、不模拟登录态（账号封禁风险）

## 二、微博

### 2.1 反爬机制分析

**IP层面：**
- 请求频率限制：每分钟超过60次请求触发验证码
- IP封禁策略：临时封禁15分钟~24小时，重复违规永久封禁
- 代理池检测：已能识别数据中心代理IP

**账号层面：**
- Cookie有效期：通常7天，需自动刷新机制
- 登录态检测：Token过期立即重定向登录页
- 多账号关联：同一IP登录多个账号会被标记

**行为层面：**
- 验证码触发：频繁操作、异常IP、非正常User-Agent
- 验证码类型：滑动验证码（主要）、图形验证码

**JS层面：**
- 请求签名：某些API需要XHB参数（类似csrf token）
- 浏览器环境检测：navigator对象、webgl指纹

### 2.2 应对策略

```python
# 方案：Cookie池 + 住宅代理 + 频率控制
import requests
import time
import random

COOKIE_POOL = [...]  # 20+ Cookie池
PROXY_POOL = [...]    # 住宅代理池

def fetch(uid, page=1):
    cookie = random.choice(COOKIE_POOL)
    proxy = random.choice(PROXY_POOL)
    
    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
        'Referer': 'https://weibo.com/',
        'X-Requested-With': 'XMLHttpRequest'
    }
    
    # 频率控制：随机延时1-3秒
    time.sleep(random.uniform(1, 3))
    
    resp = requests.get(
        f'https://weibo.com/ajax/statuses/mymblog',
        params={'uid': uid, 'page': page},
        headers=headers,
        cookies=cookie,
        proxies={'http': proxy},
        timeout=10
    )
    return resp.json()
```

### 2.3 合规建议

- ✅ Robots.txt：允许部分路径，需遵守
- ❌ 不要高频轰炸API，会触发法律风险
- ⚠️ 2019年微博起诉币乎有判例，注意频率

## 三、微信公众号

### 3.1 反爬机制分析

**核心难点：**
- 微信公众号没有公开API
- 搜狗搜索是唯一合规入口
- Cookie有效期短（24小时内）

**反爬机制：**
- 搜狗反爬：IP频率限制、验证码
- 微信公众号本身的访问需要登录态
- 历史消息列表需要更多Cookie字段

### 3.2 应对策略

```python
# 方案：搜狗搜索 + Cookie池轮换
import requests

SEARCH_URL = 'https://weixin.sogou.com/weixin'
TOKEN_URL = 'https://weixin.sogou.com/weixinopen'

def search_gzh(keyword, cookie_pool):
    cookie = random.choice(cookie_pool)
    proxy = random.choice(PROXY_POOL)
    
    headers = {
        'User-Agent': 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)',
        'Cookie': cookie,
        'Referer': 'https://weixin.sogou.com/'
    }
    
    resp = requests.get(
        SEARCH_URL,
        params={'type': 1, 'query': keyword, 'ie': 'utf8'},
        headers=headers,
        proxies={'http': proxy}
    )
    
    # 解析搜狗token，构造下一跳URL
    return extract_sogou_token(resp.text)
```

### 3.3 合规建议

- ✅ 使用搜狗搜索接口，合规
- ❌ 不要尝试绕过微信公众号登录态
- ⚠️ 公开文章和私有文章有本质区别

## 四、B站（视频/直播）

### 4.1 反爬机制分析

**视频内容：**
- User-Agent检测
- IP频率限制（每分钟60次）
- Referer验证（必须从bilibili.com跳转）

**直播弹幕：**
- WebSocket协议：blivedm等库已完整实现
- 弹幕风暴：热门直播间每秒上千条
- 开播状态：需要主动轮询直播间API

### 4.2 应对策略

```python
# 方案：blivedm（直播）+ API签名（视频）
# blivedm 已完整封装WebSocket协议，直接使用

from blivedm import BLiveClient

async def main(room_id):
    client = BLiveClient(room_id)
    client.start()
    
    try:
        async for message in client.loop:
            if message.msg_type == 'danmaku':
                print(f"[{message.uname}] {message.content}")
    finally:
        client.stop()

# 视频弹幕（HTTP XML）
# https://api.bilibili.com/x/v1/dm/list.so?oid={avid}
```

### 4.3 合规建议

- ✅ B站有明确API支持弹幕获取
- ✅ Robots.txt 相对宽松
- ⚠️ 直播弹幕抓取需注意不要干扰直播间

## 五、小红书

### 5.1 反爬机制分析

**JS混淆：**
- 请求参数使用AES + Base64签名
- `x-s`、`x-t` 等动态token
- JS代码高度混淆

**设备指纹：**
- 设备ID（_unique_id）生成算法
- 屏幕分辨率、时区、语言
- WebGL渲染指纹

**行为检测：**
- 请求顺序检测（非正常顺序会被封）
- 验证码触发频繁

### 5.2 应对策略

```python
# 方案A：浏览器自动化（Playwright）
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()

    page.set_extra_http_headers({
        'User-Agent': 'Mozilla/5.0...',
        'Accept-Language': 'zh-CN,zh;q=0.9'
    })
    
    page.goto('https://www.xiaohongshu.com/...')
    # 等待JS执行完成
    page.wait_for_timeout(3000)
    
    content = page.content()

# 方案B：JS逆向（高难度）
# 需要逆向x-s签名算法，工作量大，不建议Phase 1投入
```

### 5.3 合规建议

- ⚠️ 小红书服务条款明确禁止爬虫
- ⚠️ 建议先申请官方API合作

## 六、抖音/快手

### 6.1 反爬机制分析

**设备指纹：**
- 设备ID（device_id）算法复杂
- 必须带有真实设备签名参数

**短链接重定向：**
- 抖音分享链接通常是短链接，需解析真实URL
- 真实请求需要携带真实设备参数

**API签名：**
- 请求需带 `ms.Token`、`Xg-Token` 等
- 签名算法每版本更新

### 6.2 应对策略

```python
# 方案：分布式 + 设备指纹池
# 需要大量真实设备数据，建议外包或采购第三方数据

# 替代方案：抖音官方数据服务
# https://open.douyin.com/platform
```

### 6.3 合规建议

- ❌ 抖音/快手对爬虫态度强硬，法律风险高
- ⚠️ 建议通过官方API申请渠道获取数据
- ⚠️ 搬运/存储内容有版权风险

## 七、知乎

### 7.1 反爬机制分析

**API限流：**
- 未登录用户：60次/小时
- 登录用户：500次/小时
- 验证码触发后降低至10次/小时

**登录态检测：**
- Token过期需重新登录
- 账号异常会触发短信验证

### 7.2 应对策略

```python
# 方案：API申请 + Token轮换
ZHIHU_COOKIES_POOL = [...]

def fetch_zhihu(url):
    cookie = random.choice(ZHIHU_COOKIES_POOL)
    
    headers = {
        'User-Agent': 'Mozilla/5.0...',
        'Cookie': cookie,
        'Referer': 'https://www.zhihu.com/'
    }
    
    resp = requests.get(url, headers=headers, timeout=10)
    
    # 检测登录态
    if 'login' in resp.url:
        # 切换新Cookie
        return fetch_zhihu(url)
    
    return resp.json()
```

### 7.3 合规建议

- ✅ 知乎robots.txt相对友好
- ✅ 官方有API申请渠道，建议优先申请
