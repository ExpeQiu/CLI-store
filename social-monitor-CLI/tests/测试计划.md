# social-monitor 逐渠道测试计划

**基础设施约定**：存储与缓存统一使用本地 **PostgreSQL + Redis**，不使用 JSON 文件 / MySQL 作为测试存储。

**监控范围**：微博、微信公众号、B站、抖音、小红书（知乎等非列渠道不在日常 monitor 调度内）。

按「基础设施 → 监控编排 → 五平台渠道 → 横切能力」顺序推进。

---

## 0. 环境与基线

### 0.1 本地服务

```bash
./scripts/start.sh          # postgres + redis + rsshub
./scripts/init-db.sh
social-monitor config init
social-monitor monitor init   # 复制 monitor.yaml
```

### 0.2 监控编排冒烟

| 步骤 | 命令 | 通过标准 |
|------|------|----------|
| 干跑 | `social-monitor monitor run --task daily --dry-run` | 输出各平台 task 列表 |
| 日任务 | `social-monitor monitor run --task daily` | JSON summary 无 fatal error |
| 直播检测 | `social-monitor monitor run --task live --dry-run` | 含 bilibili/douyin live 任务 |

### 0.3 cron 示例

```cron
0 8 * * * /path/to/scripts/monitor-daily.sh
*/30 * * * * /path/to/scripts/monitor-live.sh
```
