# Comfyui-CLI 使用指南

Comfyui-CLI 是完全封装 ComfyUI 的命令行工具，将底层节点图抽象为**能力 + 工作流 + 参数**，支持：

- **图片生成**（文生图 / 图生图）
- **视频生成**（文生视频 / 图生视频）
- **数字人**（口播合成 / 带货多步链）

你无需了解 ComfyUI 节点 ID，只需通过 `inject` 字段配置生成参数。

---

## 1. 安装与初始化

### 1.1 环境要求

- Python ≥ 3.11
- 本地 ComfyUI 服务（默认 `http://127.0.0.1:8188`）
- 数字人云端能力需 RunningHub API Key（可选）

### 1.2 一键安装

```bash
cd /Volumes/Lexar/git/15CLI/Comfyui-CLI
./setup.sh        # 创建 .venv、安装包、生成 config.yaml
./verify.sh       # 验证 ComfyUI 连通性与工作流清单
```

### 1.3 全局配置初始化（可选）

若希望在任意目录使用 CLI，可初始化用户级配置：

```bash
comfyui init --cwd          # 自动绑定当前项目为 root
comfyui init --force        # 覆盖已有 ~/.comfyui/config.yaml
```

配置发现优先级：

```
--root 显式指定 → COMFYUI_ROOT 环境变量 → 当前目录向上查找 → ~/.comfyui → 包目录
```

## 2. 配置文件

主配置文件为项目根目录的 `config.yaml`（由 `config.example.yaml` 复制而来）。

### 2.1 核心配置项

| 配置块 | 说明 |
|--------|------|
| `comfyui.endpoint` | ComfyUI API 地址 |
| `comfyui.default_profile` | 默认生成档位（`fast` / `default`） |
| `capabilities.*` | 能力 → 工作流名绑定 |
| `profiles.*` | 分辨率、步数、帧数等预设 |
| `providers.runninghub` | 数字人云端后端 |

### 2.2 环境变量覆盖

```bash
export COMFYUI_ENDPOINT=http://127.0.0.1:8188
export COMFYUI_ROOT=/path/to/Comfyui-CLI
export RUNNINGHUB_API_KEY=your_key
export LOG_LEVEL=DEBUG
```

### 2.3 查看与修改配置

```bash
comfyui config show
comfyui config validate

# 切换文生图工作流为 FLUX
comfyui config set capabilities.image.t2i flux_txt2img --save

# 修改 ComfyUI 端点
comfyui config set comfyui.endpoint http://remote:8188 --save
```

## 3. 内置工作流

| 名称 | 类别 | 能力 | 来源 |
|------|------|------|------|
| `wan2.1_txt2img` | image | t2i | 本地 ComfyUI |
| `flux_txt2img` | image | t2i | 本地 ComfyUI |
| `wan2.1_txt2vid` | video | t2v | 本地 ComfyUI |
| `wan2.1_img2vid` | video | i2v | 本地 ComfyUI |
| `digital_image` | digital | compose | RunningHub |
| `digital_combination` | digital | lip_sync | RunningHub |
| `digital_customize` | digital | customize | RunningHub |

工作流文件位置：

- 内置：`workflows/image|video|digital/`
- 用户覆盖：`data/workflows/`（**优先级更高**）

## 4. 命令速查

### 4.1 基础命令

```bash
comfyui --version
comfyui health                    # ComfyUI + 工作流就绪状态
comfyui status                    # 连通性检查
comfyui status --endpoint http://remote:8188
```

### 4.2 工作流管理

```bash
# 列出全部工作流
comfyui workflow list

# 按类别筛选
comfyui workflow list --category image

# 查看可配置字段（inject / defaults / models）
comfyui workflow inspect wan2.1_txt2img

# 校验单个或全部
comfyui workflow validate wan2.1_txt2img
comfyui workflow validate --all

# 注册自定义工作流
comfyui workflow add ./my_workflow.json --name my_t2i --category image
```

### 4.3 通用执行

按工作流名直接执行，适合高级用户或 Agent 调用：

```bash
comfyui run wan2.1_txt2img \
  --prompt "白底红苹果，产品摄影" \
  -p width=512 \
  -p height=288 \
  -p steps=10 \
  --profile fast \
  -o ./outputs
```

参数说明：

| 参数 | 说明 |
|------|------|
| `--prompt` | `positive_prompt` 简写 |
| `-p key=value` | 覆盖 inject 字段，可重复 |
| `--profile` | 使用 `config.yaml` 中的 profile 预设 |
| `-o` | 输出目录（默认 `outputs/`） |
| `--root` | 指定项目根目录 |

## 5. 图片生成

### 5.1 文生图（t2i）

```bash
comfyui image t2i -t "白底红苹果，产品摄影，高清" --profile fast -o ./outputs

# 指定工作流
comfyui image t2i -t "赛博朋克城市" --workflow flux_txt2img -o ./outputs

# 覆盖参数
comfyui image t2i -t "红苹果" -p steps=15 -p seed=42 -o ./outputs
```

### 5.2 图生图（i2i）

需先在 `config.yaml` 配置 `capabilities.image.i2i` 工作流：

```bash
comfyui config set capabilities.image.i2i my_i2i --save

comfyui image i2i \
  --input ./reference.jpg \
  -t "转为水彩画风格" \
  -o ./outputs
```

## 6. 视频生成

### 6.1 文生视频（t2v）

```bash
comfyui video t2v -t "无人机航拍城市夜景，电影感" --profile fast -o ./outputs

# 指定帧数
comfyui video t2v -t "海浪拍打礁石" --length 49 -o ./outputs
```

### 6.2 图生视频（i2v）

```bash
comfyui video i2v \
  --input ./keyframe.png \
  -t "缓慢推进镜头，电影感运动" \
  --duration 3 \
  -o ./outputs
```

说明：

- `--duration` 按秒计算帧数（自动对齐 Wan 要求的 4n+1 帧规则）
- 也可直接用 `--length 25` 指定帧数
- I2V 失败时会自动降级为 T2V

## 7. 数字人

数字人默认使用 RunningHub 云端工作流，需先启用：

```bash
export RUNNINGHUB_API_KEY=your_key
comfyui config set providers.runninghub.enabled true --save
```

### 7.1 口播合成（图 + 音频）

```bash
comfyui digital run \
  --portrait ./host.jpg \
  --audio ./speech.mp3 \
  -o ./outputs
```

### 7.2 带货多步链

```bash
comfyui digital pipeline \
  --portrait ./host.jpg \
  --product ./product.jpg \
  --script "今天给大家推荐这款产品..." \
  --audio ./speech.mp3 \
  -o ./outputs
```

> 注：RunningHub 完整 API 接入中；未配置 API Key 时命令会提示错误。本地数字人工作流可通过 `workflow add` 注册后替换 `capabilities.digital.*` 绑定。

## 8. 工作流配置机制

### 8.1 参数合并优先级

```
工作流 defaults → profile 预设 → CLI -p 参数覆盖
```

例如 `fast` profile 默认 `512×288`，CLI 传 `-p width=832` 会覆盖宽度。

### 8.2 自定义工作流

1. 在 ComfyUI UI 中调试并导出 API prompt JSON
2. 按信封格式补充 `inject`、`defaults`、`required_nodes` 等字段
3. 注册并绑定：

```bash
comfyui workflow add ./data/workflows/my_t2i.json --name my_t2i --category image
comfyui workflow validate my_t2i
comfyui config set capabilities.image.t2i my_t2i --save
```

### 8.3 工作流 JSON 信封格式

```json
{
  "name": "我的工作流",
  "category": "image",
  "capability": "t2i",
  "source": "local",
  "defaults": { "positive_prompt": "...", "width": 832, "seed": -1 },
  "inject": {
    "positive_prompt": ["6", "inputs", "text"],
    "width": ["8", "inputs", "width"]
  },
  "required_nodes": ["UNETLoader", "KSampler", "SaveImage"],
  "output_nodes": ["11"],
  "prompt": { "...": "ComfyUI API 原生节点图" }
}
```

`inject` 将语义化参数名映射到 ComfyUI 节点路径，是 CLI 可配置的核心。

## 9. 输出格式

所有命令默认输出 JSON 到 stdout，便于 Agent / 脚本解析：

```json
{
  "ok": true,
  "workflow": "wan2.1_txt2img",
  "prompt_id": "abc-123",
  "duration_sec": 42.3,
  "outputs": ["/path/to/Comfyui-CLI/outputs/image.png"],
  "params": {
    "positive_prompt": "白底红苹果",
    "width": 512,
    "seed": 12345678
  }
}
```

错误时：

```json
{
  "ok": false,
  "error": "无法连接 ComfyUI: http://127.0.0.1:8188"
}
```

## 10. 常见问题

### ComfyUI 未连接

```bash
comfyui status
# 确认 ComfyUI 已启动，检查 endpoint 配置
comfyui config set comfyui.endpoint http://127.0.0.1:8188 --save
```

### 工作流 not_ready

```bash
comfyui workflow inspect wan2.1_txt2img
comfyui workflow validate --all
```

常见原因：

- ComfyUI 离线 → 显示 `comfyui_offline`
- 缺少自定义节点 → `missing_nodes`
- 模型未下载 → `missing_models`
- 数字人云端未启用 → `cloud_enabled=false`

### 切换生成档位

```bash
# 快速预览（512×288，10 步）
comfyui image t2i -t "测试" --profile fast -o ./outputs

# 高质量（832×480，20 步）
comfyui image t2i -t "测试" --profile default -o ./outputs
```

### 调试日志

```bash
comfyui --log-level DEBUG health
```

## 13. 外部集成（CyberSpace AI）

Comfyui-CLI 可作为 CyberSpace AI 的 ComfyUI 能力中间层。

### 13.1 启动

```bash
./start-web.sh   # 默认 http://127.0.0.1:8765
```

### 13.2 关键 HTTP 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/ping` | 存活探测 |
| GET | `/api/health?full=true` | 健康检查 + workflow 就绪 |
| GET | `/api/workflows?category=image` | 工作流列表 |
| POST | `/api/image/t2i` | 异步文生图 → `{ job_id }` |
| POST | `/api/image/i2i` | multipart 图生图 |
| POST | `/api/video/t2v` | 异步文生视频 |
| POST | `/api/video/i2v` | multipart 图生视频 |
| POST | `/api/run` | 通用 workflow 执行 |
| GET | `/api/jobs/{job_id}` | 轮询任务状态/结果 |

### 13.3 Job 协议

1. `POST` 提交任务 → 获得 `job_id`
2. `GET /api/jobs/{job_id}` 轮询至 `status=completed`
3. 从 `result.output_absolute_urls[0]` 下载输出文件

### 13.4 CyberSpace AI 配置

```env
COMFYUI_CLI_ENDPOINT=http://127.0.0.1:8765
AI_IMAGE_CLIENT=comfyui-cli
AI_VIDEO_CLIENT=comfyui-cli
```

验证：`node scripts/diagnostics/verify-comfyui-cli.mjs`（在 cyberspaceAI 项目中）
