# 剪映专业版工程格式调研

> Phase 4 结论：剪映 **draft 工程为私有格式**，暂无稳定公开的导入 API。Video-editing-CLI 采用 **元数据 JSON** 作为过渡方案。

## 现状

| 格式 | 支持情况 | 说明 |
|------|----------|------|
| FCPXML | ✅ 完整 | 达芬奇 / FCP 推荐 |
| EDL (CMX3600) | ✅ 完整 | Premiere / Avid |
| SRT | ✅ 完整 | 通用字幕 |
| 剪映 draft | ❌ 未开放 | 目录含 `draft_content.json` 等私有结构 |

## 剪映 draft 结构（观测）

```
draft/
├── draft_content.json    # 时间轴、素材引用（加密/版本敏感）
├── draft_meta_info.json
└── Resources/            # 媒体缓存
```

- 版本升级后字段频繁变化
- 无官方 SDK 写入 timeline 切点

## Video-editing-CLI 方案

导出 `jianying_metadata.json`（`export.formats` 含 `jianying` 时）：

- 源视频绝对路径
- 保留片段 `source_in` / `source_out`
- 手动导入步骤说明

**推荐工作流：**

```
A-Roll 初剪 → FCPXML/EDL → 达芬奇精剪 → 导出 MP4 → 剪映加特效/字幕
```

或：

```
A-Roll 初剪 → jianying_metadata.json → 剪映手动按切点剪辑
```

## 后续可能路径
