# 视频号新能源汽车发布会监控实战清单

## 1. 目标

今天的目标是验证基于 **Playwright DOM 解析 + 凭证持久化** 的方式，能否稳定跑通一场视频号（WeChat Channels）的新能源汽车发布会监控：

- 成功扫码登录并保存凭证
- 利用凭证绕过登录限制进入直播间
- 持续抓取弹幕
- 持续记录在线人数（看过人数）快照
- 能在直播结束后拿到原始数据库结果

说明：
- 视频号网页版对风控要求极高，**请务必使用小号扫码，不要使用主微信账号！**
- 视频号的弹幕 DOM 结构可能随微信版本更新而变化，如果抓不到数据，需及时调整 `sph.py` 中的选择器。

---

## 2. 开播前准备

### 2.1 环境确认

在项目根目录执行：

```bash
./venv/bin/python main.py --help
./venv/bin/python main.py init-db
```

### 2.2 确认房间号

视频号网页版的直播间房间号一般来自直播链接，例如：

```text
https://channels.weixin.qq.com/web/live/12345678
```

其中 `12345678` 就是 `room_id`。如果是在手机微信里看到的直播，请尝试点击右上角分享 -> 复制链接，在浏览器中打开以获取准确的网页端 `room_id`。

### 2.3 明确任务元信息

建议在启动前先确定两项元信息：

- `event_name`: 例如 `2026 某品牌新车发布会-视频号`
- `car_brand`: 例如 `xiaomi`、`byd`、`nio`、`xpeng`

---

## 3. 授权与启动命令

视频号监控分为**两步走**：先获取凭证，再启动监控。

### 3.1 步骤一：扫码登录获取凭证 (只需执行一次)

执行以下命令：

```bash
./venv/bin/python main.py login sph
```

1. 终端会提示打开浏览器，随后会弹出有界面的 Chromium 浏览器。
2. 页面会跳转到 `https://channels.weixin.qq.com`，请在 **60 秒内**使用微信扫码登录。
3. 登录成功后，终端会提示：`✅ 登录状态已保存至: .../auth/sph_state.json`。
4. 浏览器会自动关闭。

### 3.2 步骤二：启动监控

获取凭证后，建议今天优先使用有头模式（`--headed`），便于观察页面是否正常加载和弹幕是否滚动：

```bash
./venv/bin/python main.py start sph <room_id> --event-name "2026 某品牌新车发布会" --car-brand "品牌名" --headed
```

示例：

```bash
./venv/bin/python main.py start sph 12345678 --event-name "2026 某品牌新车发布会" --car-brand "xiaomi" --headed
```

如果确认稳定，后续可去掉 `--headed` 使用无头模式。

---

## 4. 开播中巡检要点

### 4.1 终端应看到的正常日志

正常情况下，终端会出现类似输出：

```text
[sph] 启动采集，房间号: 12345678
[sph] 正在打开视频号直播间: https://channels.weixin.qq.com/web/live/12345678
[sph] 抓取到 15 条新弹幕
[sph] 成功将 20 条弹幕写入数据库，当前热度: 105000
```

### 4.2 人工巡检频率

视频号的风控较严，建议每 **5 到 10 分钟**检查一次：

- 浏览器页面是否还停留在直播间，**有没有弹出滑块验证或重新登录提示**。
- 终端是否仍有新增弹幕日志。
- 热度值（看过人数）是否在正常变化。

### 4.3 建议人工记录的关键时间点

与 B 站一致，建议记录开播、核心车型亮相、价格公布等关键时间点，供后续复盘。

---

## 5. 直播结束后

### 5.1 正常停止

在终端按 `Ctrl+C` 结束采集。程序会做最后一次数据落库并安全关闭浏览器。

### 5.2 快速检查数据

```bash
./venv/bin/python - <<'PY'
from app.core.database import SessionLocal
from app.models.schema import EventTask, DanmakuRecord, LiveMetric

db = SessionLocal()
try:
    print("tasks =", db.query(EventTask).count())
    print("danmakus =", db.query(DanmakuRecord).count())
    print("metrics =", db.query(LiveMetric).count())
finally:
    db.close()
PY
```

### 5.3 导出赛后统计

同样支持使用脚本快速导出基础统计：

```bash
./venv/bin/python scripts/post_event_summary.py
```

## 6. 当前已知限制与风险预警

- **强依赖 DOM 结构**：视频号的 Web 端更新较频繁，如果在 `--headed` 模式下看到弹幕在动，但终端没有抓取日志，说明 DOM 结构变了，需要立刻修改 `app/scraper/sph.py` 里的 `evaluate` 脚本。
- **账号风控风险**：长时间高频抓取可能会被微信判定为机器行为。目前的采集频率是每 2 秒一次，若被踢出登录，请重新运行 `login sph`。
- **去重逻辑限制**：目前依靠 `用户名+内容` 进行哈希去重，如果用户昵称重复度高且发送内容一样（如疯狂刷“遥遥领先”），可能会被误判为重复弹幕被过滤掉一部分。

---

## 7. 异常排查

### 7.1 提示未找到登录凭证

如果终端提示：`[sph] 警告: 未找到登录凭证...`
说明没有执行 `login` 命令，或者 `auth/sph_state.json` 被删除了。请先退出程序，执行 `python main.py login sph` 重新扫码。

### 7.2 页面一直卡在登录扫码界面

说明 `sph_state.json` 里的凭证过期了（通常微信网页版 Cookie 有效期在几天到几周不等）。
解决方案：删除 `auth/sph_state.json`，重新执行 `login` 命令扫码。

### 7.3 有弹幕日志但热度始终为 0

视频号网页版左上角的“xxx人看过” DOM 结构可能改变。需要检查 `sph.py` 中的 `_get_online_count` 方法里的正则表达式和选择器。

---

## 8. 今天的推荐执行顺序

1. 准备一个专用于测试的**微信小号**。
2. 找到一场正在直播的视频号直播，获取网页版链接和 `room_id`。
3. 执行 `./venv/bin/python main.py login sph`，使用小号扫码。
4. 执行 `./venv/bin/python main.py start sph <room_id> --headed`。
5. 观察终端输出，确认能否抓到弹幕和热度。
6. 稳定运行 15 分钟后 `Ctrl+C` 停止。
7. 检查数据库数据落盘情况。
