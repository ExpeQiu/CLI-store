# Vplatform-CLI 功能说明

**Vplatform-CLI 2.1** 是精简版 **分镜驱动短视频编排 CLI**。核心通过 ComfyUI 完成 AI 生图/生视频，再自动拼接为带旁白、字幕、BGM 的成片。面向 OpenClaw + ComfyUI 本地或隧道部署。

---

## 核心能力：一键短视频流水线

输入主题（或已有分镜 JSON），自动执行完整链路：

| 阶段 | 功能 | 依赖 |
|------|------|------|
| `storyboard` | LLM 生成分镜脚本 | LLM API |
| `tts` | Edge TTS 旁白配音 | 无需 ComfyUI |
| `keyframe` | **文生图**（关键帧） | ComfyUI |
| `i2v` | **图生视频** | ComfyUI |
| `concat` | 片段拼接 | ffmpeg |
| `subtitle` | 生成 SRT/VTT 字幕 | — |
| `final` | 混音 + 烧录字幕 + BGM | ffmpeg |

可在任意阶段停止，例如只生成分镜或只跑到生图：

```bash
vplatform pipeline run -t "产品展示" --stop-at storyboard
vplatform pipeline run -t "产品展示" --stop-at keyframe
```

简写：`vplatform -t "主题" --stop-at storyboard` 等价于 `pipeline run`。

## 文生图（关键帧）

**支持文生图**，通过 ComfyUI 工作流实现，内置两套方案：

| 工作流 | 说明 | 用途 |
|--------|------|------|
| `wan2.1_txt2img` | Wan 2.1 文生图 | 默认关键帧工作流 |
| `flux_txt2img` | FLUX.1-dev 文生图 | 高质量关键帧（需本地 FLUX 模型） |

相关配置（`config.yaml` → `pipeline`）：

- `keyframe_workflow`：默认 `wan2.1_txt2img`
- `flux_workflow`：默认 `flux_txt2img`
- `use_flux_keyframe`：设为 `true` 时优先 FLUX，不可用时自动降级 Wan

流水线内自动生图，或通过 `--stop-at keyframe` 只跑到生图步骤。

### 单独生图（不走流水线）

若只需生成一张图、不跑完整短视频流水线，有两种方式：

**方式一：ComfyUI 直接跑工作流**

在 ComfyUI 界面加载并执行项目内置工作流 JSON：

- `workflows/flux_txt2img.json` — FLUX.1-dev 文生图
- `workflows/wan2.1_txt2img.json` — Wan 2.1 文生图

**方式二：`vplatform image` 子命令**

封装 `ComfyUIService.t2i()`，自动路由 ComfyUI 端点并输出图片：

```bash
# Wan 2.1（默认 keyframe 工作流）
vplatform image -p "红色苹果，白底产品摄影，8k"

# FLUX 高质量
vplatform image -p "cinematic landscape" --flux

# 指定工作流与输出目录
vplatform image -p "..." -w flux_txt2img -o outputs/my_image
vplatform image -p "..." -n "blurry, watermark" --profile fast
```

输出默认写入 `outputs/images/<timestamp>/`，JSON 结果含 `path` 字段。

## 视频生成工作流

| 工作流 | 说明 |
|--------|------|
| `wan2.1_img2vid` | 图生视频（I2V，流水线默认） |
| `wan2.1_txt2vid` | 文生视频（I2V 失败时降级） |

相关配置：

- `i2v_workflow`：默认 `wan2.1_img2vid`
- `t2v_workflow`：默认 `wan2.1_txt2vid`
- `fps`：默认 16
- `profiles`：`default` / `fast` 两套分辨率与步数预设

## CLI 命令

```bash
vplatform init --cwd              # 初始化 ~/.vplatform 配置
vplatform health                  # 健康检查（ComfyUI + 工作流列表）
vplatform config validate         # 校验配置与工作流
vplatform workflow list           # 列出可用工作流
vplatform workflow validate <name> # 校验工作流文件
vplatform status                  # ComfyUI 连通性检查

vplatform image -p "提示词" [--flux] [-w <workflow>] [-o <dir>]  # 单独生图
vplatform pipeline run -t "主题" [--stop-at <stage>] [--profile fast|default]
vplatform task list
vplatform task status <task_id>
vplatform task cancel <task_id>
```

工作流文件位于 `workflows/`（内置）与 `data/workflows/`（用户自定义），后者优先级更高。

## Web 控制台

```bash
./start.sh          # 后台启动，默认 http://127.0.0.1:8768/
./start.sh --fg     # 前台启动
./stop.sh           # 停止
```

提供轻量 REST API 与 Web 页面：

| 接口 | 说明 |
|------|------|
| `GET /api/v1/health` | 健康检查 |
| `POST /api/v1/pipeline` | 异步提交流水线任务 |
| `POST /api/v1/pipeline/sync` | 同步执行流水线 |
| `GET /api/v1/tasks/{task_id}` | 查询任务状态 |
| `GET /outputs/...` | 访问输出文件 |

API 文档：`http://127.0.0.1:8768/docs`

## 辅助能力

### LLM 分镜

- 使用 OpenAI 兼容 API 生成结构化分镜（画面词、动作词、旁白等）
- 提示词模板：`prompts/storyboard.txt`
- 支持导入已有分镜：`--storyboard-file path/to/storyboard.json`

### TTS 旁白

- 默认 Edge TTS（`zh-CN-XiaoxiaoNeural`）
- 配置项：`tts.provider`、`tts.voice`

### 字幕

- 根据分镜与 TTS 时长生成 SRT / VTT
- 最终成片烧录硬字幕

### 素材降级

ComfyUI 生图/生视频连续失败时，可降级为 Pexels 素材库视频：

- 配置 `material.pexels_api_keys`
- `material.fallback_after_failures` 控制触发阈值（默认 2 次）

### 算力路由

按优先级自动选择 ComfyUI 端点（`provider.priority`）：

1. `autodl` — AutoDL SSH 隧道（`provider.autodl_tunnel_script`）
2. `local` — 本地 ComfyUI（`comfyui.endpoint`）
3. `commercial` — 商用 endpoint（`provider.commercial_endpoint`）

详见 `guide/算力服务器.md`。

### OpenClaw 集成

```bash
./scripts/sync_openclaw_skill.sh   # 同步 Skill 到 OpenClaw
./scripts/openclaw.sh pipeline run -t "主题" --stop-at storyboard
```

环境变量 `VPLATFORM_ROOT` 由 `openclaw.sh` 自动设置。

## 输出目录

每次任务在 `outputs/<task_id>/` 下生成：

```
outputs/<task_id>/
├── storyboard.json    # 分镜（含各帧 prompt、音频、路径）
├── audio/             # TTS 旁白音频
├── keyframes/         # 关键帧图片
├── clips/             # 视频片段
├── concat.mp4         # 拼接视频
├── subtitles.srt/vtt  # 字幕
├── muxed.mp4          # 混音后视频
└── final.mp4          # 最终成片
```

任务日志：`logs/tasks/<task_id>/`
